    Protected Information Setup - Instructor Demo

    Birth dates, especially the full birth date, have been determined to be PII. The administrator (instructor) has been asked to find and hide all birth dates within the dataset.

    Apply PII Classification

    1. Click Data Catalog.

    2. Search for all of the birth date fields. Click in the search box and enter *birth*. Four fields with birth in the name are displayed.

    3. Click rider_birth_date in the list of items.

    4. The details for the rider_birth_date are displayed. Click Add Classification.

    5. Click in the Search and Add field.

    6. Enter PII and click Create.

    7. Enter PII for Name. Enter Personally Identifiable Information for Description. Click Create.

    8. The new PII classification is created. However, it will not show as a choice yet. Return to the search results by clicking Search in the left side menu. Then, click rider_birth_date. Click Add Classification and search for PII.

    9. Click the plus symbol next to PII.

    10. Click Save.

    11. Return to the search results and repeat the process to add the PII classification the the remaining three birth date fields.

    Apply Ranger Policies and Masking

    Once the fields have been tagged as PII. Ranger is used to limit access or mask the fields.

    1. Return to the Data Catalog Search page. Click the Ranger button.

    2. Select Tag Based Policies from the Access Manager menu.

    3. Click the cm_tag link.

    4. Select the Masking tab.

    5. Click the Add New Policy button.

    6. Enter Mask PII for the Policy Name.

    7. Select PII for the TAG.

    8. Under Mask Conditions, select public for Select Group.

    9. Click Add Permissions under Add Access Types**.

    10. Select hive as the component.

    11. Click the Hive checkbox.

    12. Click the checkmark button.

    13. Click Add Mask Type under Select Masking Options.

    14. Select Custom.

    15. Click the checkmark button.

    16. Enter cast(mask(cast(to_date({col}) as date), 'x', 'x', 'x', -1, '1', 1, 0, -1) as timestamp) into the custom mask field.

    Note

    You may wonder why Custom was chosen instead of Date show only year. The later only works with date fields. The birth date fields in the duocar database are timestamp fields. Had Data show only been chosen, the fields would have been set to Null, since the date value returned by the mask would not have matched the timestamp data type. Therefore, a custom mask is used to cast the result to timestamp.

    1. Click Add to create the new policy.

    The previous tag-based policy is sufficient to mask all of the date fields. The custom mask will return the year. The day and month will be set to 01.

    However, an additional resource policy will be created to demonstrate how resource policies work, how policies interact, and the potential effect of a policy on propagated classifications. The following steps create a new resource-based Ranger policy that specifically masks the riders table birth_date field by returning NULL.

    1. Select Resource Based Policies from the Access Manager menu.

    2. Click the Hadoop SQL link.

    3. Select the Masking tab.

    4. Click the Add New Policy button.

    5. Enter Mask Rider Birth Date for the Policy Name. Select duocar as the Hive Database.

    6. Select riders as the Hive Table.

    7. Select birth_date as the Hive Column.

    8. Click Add Permissions under Access Types.

    9. Check select and click the checkmark button.

    10. Click the Select Masking Option link under Select Masking Options.

    11. Select Nullify and click the checkmark button.

    12. Select public under Select Group.

    13. Click Add.
